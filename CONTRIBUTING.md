# Contributing to FBEP

## Rule

Documentation comes before implementation.

## Workflow

1. Read START_HERE.md first.
2. Audit the current repository state.
3. Document missing or outdated information.
4. Implement only after documentation is clear.
5. Update docs/foundation/CHANGELOG.md for every change.
6. Keep secrets out of Git.

## Branching

Use feature or foundation branches.  
Do not commit directly to main unless explicitly approved.

## Quality

All changes must preserve:

- security
- compliance
- documentation consistency
- repository structure
- version history
